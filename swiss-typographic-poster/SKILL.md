# Swiss Typographic Poster

把一个标题 + 用途，编译成**一条**确定性的、可直接交给图像模型的瑞士国际主义海报提示词。

核心原则：**选项只存在于推理里，绝不出现在输出里。** 不要给用户列「红/蓝/黄」式菜单造成误导——内部按意图选定每一个模块，输出一条解析干净的扁平提示词。

## 先读

按以下顺序读取，再开始编译：

- `references/compile-contract.md`：编译契约（消歧义的 5 条铁律 + 锁死的风格内核 + 固定字段顺序）。**每次输出前必须满足。**
- `references/module-library.md`：模块库（锚点 / 主视觉 / 配色 / 标题位置 / 画幅 / 文字模式）和意图→模块映射表。
- `references/presets.md`：6 套已编译预设。当用户只想「拿来即用」或不愿描述时，直接给最贴近的一套。

## 输入

只收两项，其余全部推断：

1. **标题**（必需）——中文主题词或一句短标题。缺失时只问这一个问题。
2. **用途**——「展示图 / 作品集」还是「传播封面」。未说明时按标题语气推断，并在选型理由里点明假设。

不要求用户填表，不展示模块选项。用户若主动指定了配色、画幅、锚点设计师等，以用户为准（覆盖推断）。

## 工作流

1. **解析意图**：从标题和上下文推断 内容类型（方法论/工具/商业/设计/决策…）、语气（理性可信/高能警示）、平台与画幅。
2. **选配模块**：按 `module-library.md` 的映射表，给每个槽位（锚点·主视觉·配色·标题位置·画幅·文字模式）**各定一个值**。拿不准时择优选定，不要并列。
3. **编译**：按 `compile-contract.md` 的字段顺序与 5 条铁律，组装成一条扁平提示词——解析掉所有选项、零中文注释、否定只进 AVOID、只声明一次要渲染的文字。
4. **输出**：一条成品提示词 + 一行「选型理由」（说明推断了什么用途/内容类型，选了哪套模块）+ 一行指向最接近的预设（备选）。

## 意图 → 模块映射（确定性判断层）

详表见 `references/module-library.md`。要点：

- **用途**：展示图/作品集 → 文字模式 TC（拉丁主标+中文小标）或 TA（纯拉丁）；传播封面 → TB（中文主标+拉丁小字，中文必须可读）。
- **内容类型**：方法论/系统/知识 → 纯网格 或 色带；工具/科技/AI → 同心弧 或 旋转方阵；商业/报告/数据 → 纯网格 或 色带；设计/字体/排版 → 字体即图 或 旋转方阵；设计实验/文化 → 受控破格；决策/层次结构 → 色带。
- **语气**：理性可信 → 钴蓝 / 纯黑白；高能警示 → 黄黑 / 红。
- **锚点跟随主视觉**：弧/网格→Müller-Brockmann，方阵/色带→Max Bill，字体→Emil Ruder，单个大形/黑白形态→Armin Hofmann，破格→Wolfgang Weingart。

## 输出格式

除「缺标题时问一个问题」外，最终输出：

1. 一条完整、可直接复制的英文骨架提示词（标题用引号/「」声明，符合编译契约）。
2. 一行选型理由。
3. 一行：最接近的预设编号（来自 `presets.md`），供用户想要更稳时直接换用。

不要输出模块菜单、多版本选项、参数表、推理过程或自检记录。

## 质量门槛

- 输出里**零斜杠选项、零大括号、零中文注释**（标题文字除外）。
- **一个主视觉**，不并列两种。
- 否定词只在 AVOID 块，正文全正向。
- 风格内核（统一网格 / 单一 grotesk / 平涂硬边无渐变3D纹理 / 客观无装饰 / 非对称倾向）每条必带。
- 不编造不存在的生图结果；本 skill 只产出提示词。
