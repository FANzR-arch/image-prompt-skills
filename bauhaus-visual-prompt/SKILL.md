# Bauhaus Visual Prompt Skill

把一个标题、主题、用途或室内参考图，编译成**一条**可直接交给图像模型的包豪斯视觉提示词。

核心原则：**先判断媒介，再调用规则。** 包豪斯不是红黄蓝几何装饰；不同媒介必须拆成不同的执行重点：封面看版式，配图看概念结构，海报看图文冲击，室内重绘看空间、家具、材质和动线。

## Read

按以下顺序读取，再开始编译：

- `references/compile-contract.md`：编译契约、字段顺序、输出质量门槛。每次输出前必须满足。
- `references/module-library.md`：用途、画幅、主视觉、色彩、文字模式、室内重绘规则的模块库。
- `references/presets.md`：4 套已编译预设。用户只想拿来即用或输入很少时，直接选最贴近的一套改写。

## Input

优先从用户话语里推断，不要求填表：

1. **用途**：文章封面、正文配图、栏目/活动海报、室内照片重绘。缺失时按上下文推断；无法推断时只问这一个问题。
2. **标题 / 主题**：封面和海报通常需要；正文配图可用概念；室内重绘可不需要标题。
3. **画幅**：未说明时，封面默认 4:5，正文配图默认 16:9，海报默认 2:3，室内重绘沿用参考图或 16:9。
4. **参考图**：如果用户上传室内照片，优先走室内重绘，保留原图空间结构。

用户若主动指定画幅、主标题、配色、是否保留文字等，以用户为准；但不能破坏风格纯度和媒介边界。

## Workflow

1. **解析意图**：判断 use case：`article-cover`、`inline-illustration`、`poster`、`interior-redesign` 四选一。
2. **选配模块**：按 `module-library.md` 给每个槽位定一个值：画幅、主视觉、色彩、文字模式、材质/空间规则、禁止项。
3. **编译提示词**：按 `compile-contract.md` 的字段顺序输出一条英文美术指导。解析掉所有 `{}`、`/` 选项和中文注释；标题文字例外。
4. **输出**：一条成品提示词 + 一行选型理由 + 一行最接近预设。不要输出模块菜单、推理过程、自检清单或多版本选项。

## Routing Rules

- **文章封面**：标题区、几何版式、视觉重心、留白、短文本控制。
- **正文配图**：概念说明、结构图、几何关系、低噪音、少文字或无文字。
- **栏目/活动海报**：展览海报感、强标题、图文一体化、较强构图张力。
- **室内照片重绘**：保留原图视角、比例、门窗、墙体和主要家具位置；只改材质、家具语言、灯具、色彩和空间气质。

## Output Format

除非缺少必要输入需要追问，最终只输出：

1. 完整成品提示词。
2. `选型理由：...`
3. `最接近预设：...`

## Usage Examples

这些请求应直接触发本 skill：

```text
生成一张 4:5 包豪斯风格文章封面，标题是：结构比装饰更重要。
```

```text
生成一张 16:9 包豪斯风格正文配图，主题是：提示词不是描述风格，而是组织结构。
```

```text
生成一张 2:3 包豪斯风格活动海报，标题是：FORM FOLLOWS FUNCTION。
```

```text
参考我上传的室内照片，把这个房间重绘成包豪斯风格工作室。
```

## Quality Gates

- 输出里不能保留 `{变量}`、斜杠菜单、中文注释或多套方案。
- 每次只选一个主视觉，不并列多个引擎。
- 否定项集中放在 `AVOID`。
- 不把海报排版规则塞进室内重绘，不把家具、玻璃幕墙塞进文章封面。
- 室内重绘必须明确保留原始空间结构；不能把参考照片完全重画成无关样板间。
- 不编造生图结果；本 skill 只产出提示词。
