# Acid Depth Poster（前进色 × 后退色实验海报）

把用户给的主题、文章、标题或情绪，编译成**一条**可直接交给图像模型的实验海报提示词。风格核心是「清晰前景层 ＋ 模糊后景层」的空间结构，气质落在实验杂志封面 / 青年文化海报 / 地下传单 / Y2K / Acid Graphics 之间。

核心原则：**模板正文锁死，输入区放开。skill 的全部工作在于填好输入区；正文一字不改地照抄，创作自由留给图像模型。**

## Read

- 生成提示词：读 `references/master-template.md`（唯一真源模板）。

## Input

优先从用户输入推断，不要求填表。输入区共 11 个字段，只有「主题」必填：

| 字段 | 填写判断 |
|------|---------|
| 主题 | 必填。从文章、标题或用户描述提炼成一句话，带情绪指向，不是干巴巴的分类词 |
| 主标题 | 用户给了标题就原文照抄，一个字不改；没给且能从文章提出强标题时填 2-8 字短标题；把握不大就留空 |
| 副标题 | 有现成的填，没有留空 |
| 核心关键词 | 从主题提 2-4 个，帮图像模型抓氛围；不确定就留空 |
| 辅助文字 | 有具体短句（观点句、日期、编号）才填；不要为填而编 |
| 英文短句或标签 | 有自然贴合的才填（如产品名、年份、slogan）；不要硬造装饰英文 |
| 配色倾向 | 用户指定才填；否则留空让模型按主题选 |
| 画面氛围 | 能从主题读出明确情绪时填（如：深夜赶稿的躁动、发布前的兴奋）；模糊就留空 |
| 照片或场景倾向 | 主题有具体场景锚点时填（如：深夜办公室、便利店、网吧）；否则留空 |
| 画幅比例 | 优先用户指定；按用途推断：X/公众号封面横版 16:9 或 5:2，海报竖版 3:4 或 2:3；完全没有线索就留空 |
| 文字语言 | 默认中文为主；用户要英文版才改 |

留空是合法输出，不是偷懒：模板明确规定留空字段由图像模型自由判断。**宁可留空，不要编造**——填一个错误的具体值（假日期、硬凑的英文、无根据的配色）比留空更伤画面。

## Workflow

1. **识别输入**：判断是单张生成、成组生成，还是用户只要空白模板。
2. **提炼主题**：把文章或描述压成一句有情绪的主题句。用户给了确切文字（标题、副标题、短句）必须原文保留，不翻译、不润色。
3. **填输入区**：按上表逐字段判断填或留空。
4. **组装输出**：填好的输入区 + 填写说明 + 模板固定正文（从【生成任务】到【最终效果】，原样照抄）+ 负面控制词（默认附加；用户说「不要负面词 / 精简版」时省略）。整体放进一个 ```text 代码块，复制即用。
5. **成组生成**：每张单独出一条完整提示词，输入区至少在「配色倾向、画面氛围、照片或场景倾向」中变化两项，避免整组同质化。

## Output Format

````markdown
## 已填输入区
- 主题：…
- 主标题：…（留空字段标「留空→模型自判」）
- 画幅比例：…

## 完整提示词
```text
【用户输入区】

主题：…
主标题：…
（11 个字段全部列出，留空的字段保留字段名、值为空）

填写说明：
除“主题”外，其余内容均可留空。留空的部分由模型根据主题和整体视觉方向自由判断与补充。

【生成任务】
…（固定正文原样到【最终效果】结束）

【负面控制】
避免所有元素处于相同清晰度，…
```

## 填法理由
一两句：主标题为什么这样定、场景/配色为什么填或留空。
````

用户只要 prompt 时省略「已填输入区」和「填法理由」，只给代码块。用户要空白模板时，直接输出 `master-template.md` 里的输入区 + 固定正文 + 负面控制词，不做任何填写。

## Quality Gates

- 固定正文必须与 `master-template.md` 完全一致：不删段、不缩写、不换措辞、不调整段落顺序。模板正文本身就是经过设计的 prompt，改动即劣化。
- 用户指定的文字（主标题、副标题、辅助文字）必须原文出现在输入区，不翻译、不改字。
- 不为留空字段编造具体值；不生成假日期、假地点、假编号（除非主题真实包含）。
- 输出必须是一条完整独立的提示词，用户复制代码块即可生图，不需要再看本 skill 或补任何内容。
- 成组输出时每条提示词都完整自足，且输入区有明显差异。
- 本 skill 只产出提示词，不编造已生成的图片结果。
