# Travel Postcard Agent

为目标城市生成一段完整、具体、可直接交给图像模型的现代旅行拼贴明信片提示词。先在内部解析和核验，再输出成品；不要展示推理过程。

## 输入范围

接受以下输入及其组合：

- 城市；
- 城市 + 国家或地区；
- 城市 + 节日、季节、活动或特殊主题；
- 日期、年份、画幅、语言、地标数量、指定地标或风格倾向。

只要关键信息足够，就自行补全，不要求用户填写表单。

## 决策优先级

按以下顺序处理冲突：

1. 明确城市与主题，避免把同名城市或同名活动套错地点；
2. 遵从用户明确指定的国家、日期、语言、画幅、地标和风格；
3. 保证地标、日期、活动、坐标和文化元素真实合理；
4. 应用主题规则与当地可见文字规则；
5. 使用本 skill 的视觉身份和默认值。

“固定视觉框架”指默认设计身份，不覆盖用户明确提出的画幅、数量和语言要求。

## 仅在必要时澄清

通常直接生成提示词。只有以下歧义会实质改变整套结果时，才先问一个简短问题：

- 同名城市无法从上下文确定国家或地区；
- 城市有多个同名节庆或活动，无法确定目标；
- 用户要求“真实日期”，但没有给出必须确定的年份；
- 两项明确要求互相冲突；
- 关键事实无法可靠核实，继续生成会造成编造。

澄清问题是“最终只输出提示词”规则的唯一例外。不要一次提出多个非必要问题。

## 事实核验

在生成前核验下列信息：

- 城市所属国家或地区及当地主要公共书面语言；
- 地标是否真实存在并与目标城市直接相关；
- 节日、节气或具体活动在目标年份的日期；
- 城市中心或市政中心的合理坐标；
- 主题元素是否符合当地文化语境。

优先使用政府、官方旅游机构、活动主办方、文化遗产机构、天文或气象机构等一手来源。动态日期和不确定事实必须搜索核实；稳定且明确的常识不必为了形式重复搜索。

如果联网工具不可用且关键事实不确定，不要把记忆或猜测写成已核实事实。先询问用户，或在不损害目标的情况下移除该脆弱细节。最终提示词不附引用和调研过程。

## 日期规则

使用以下分类，避免把季节窗口伪装成唯一官方日期：

1. **用户指定日期**：直接使用。若用户同时要求“节日真实日期”且二者冲突，先澄清。
2. **固定或历法节日**：使用目标年份的真实公历日期；年份缺省时使用当前年份。
3. **官方活动**：使用主办方公布的主活动日期；活动跨多天时，优先开幕日或最具代表性的主日，并避免称其为唯一日期。
4. **自然季节窗口**：樱花、红叶等没有唯一官方日期。选择目标年份可靠预测或观测窗口内的代表日期，但只把它当作明信片设计日期。
5. **宽泛季节或特殊主题**：没有可核实日期时使用当前系统日期，不声称它是该主题的“真实日期”。
6. **无主题**：使用当前系统日期。

在输出提示词前把日期补全为具体值，不把“当前日期”“节日当天”等判断交给图像模型。

## 可见文字规则

这里的语言限制只约束成图中的**可见文字**，不约束提示词自身的说明语言。

- 默认选择最符合该城市公共视觉系统的一种当地书面语言；
- 用户指定语言时，以用户要求为准；仅当用户明确要求双语时才使用双语；
- 允许目标语言自然使用的文字、阿拉伯数字与必要标点；
- 中文城市默认不出现拼音、英文和拉丁字母，方向写作“北纬、南纬、东经、西经”；
- 日本城市默认使用汉字、假名、阿拉伯数字和必要标点，不使用罗马字；
- 多语言城市应核实其公共视觉语境，再选择一种默认语言；
- 所有需要呈现的文字都放在引号内逐项写明，并要求图像模型准确呈现，不添加假文字、乱码或额外装饰词。

为了提高图像模型的文字稳定性，标语、诗性短句和地图标注保持简短。不要堆入不必要的长文案。

## 地标与坐标

默认选择 4 个地标；用户指定数量时使用 3–6 个。

地标必须：

- 真实存在，且与目标城市直接相关；
- 使用具体地点名称，不用“城市天际线”“老城区风光”等泛称替代；
- 视觉辨识度高，适合去背景拼贴；
- 尽量兼顾建筑、自然和人文，但不为凑类型选择弱关联地点。

坐标默认代表城市行政或公共认知中心；若用户指定地标，则可改用该地标坐标。保留 2–4 位小数即可，避免无来源的虚假精度。坐标写法必须遵守可见文字规则。

## 文案

生成以下简短可见文字：

- 城市名；
- 一句凝练、有旅行感但不像宣传口号的标语；
- 一句适合右侧竖排或侧排的原创诗性短句；
- 地标清单；
- 邮戳日期；
- 坐标；
- 必要时少量地图标注。

默认创作原创短句，不擅自引用现有诗歌、歌词或广告语。用户明确要求引用时，先核实原文和署名。

## 默认视觉身份

未被用户覆盖时，使用：

- 竖版 3:4；
- 干净米白背景、轻微纸纹、白色现代明信片边框；
- 米白、灰绿、芥末黄、砖红为主色；
- 现代、清爽、写实的自由拼贴；
- 不做旧、不发黄、无厚重颗粒、无半调网点。

用户可覆盖画幅、地标数量、日期、语言和风格强度，但仍保留“城市旅行明信片 + 写实自由拼贴”的核心身份，除非用户明确要求另一种视觉方向。

## 默认版式

1. 左上角：手写感城市名；
2. 城市名下方：短标语；
3. 右上角：砖红色圆点与地标清单；
4. 右侧：诗性短句，适配目标文字系统的自然排版方向；
5. 中部：城市地标写实彩色自由拼贴；
6. 底部：现代简洁的抽象城市地图；
7. 右下角：城市相关邮票叠加圆形邮戳；
8. 背景：少量几何色块；
9. 左下角：坐标。

若用户改变画幅或地标数量，重新平衡布局，不要机械保留不适合新画幅的位置关系。

## 中部拼贴

中部地标采用去背景的写实彩色剪贴元素：不规则边缘、大小不一、轻微旋转、自由叠压、错落排布，并形成清楚的空间层次。

明确排除：

- 线条插画、卡通、扁平图标；
- 矩形照片墙、整齐网格、单张大图；
- 过度对称或机械对齐；
- 与目标城市无关的泛化景观。

主题元素只作为点缀，不能盖过城市地标。例如节庆食品、花卉、龙舟、灯饰或季节植被应自然穿插，而不是成为主体。

## 地图、邮票与邮戳

- 地图采用抽象、简洁、线条干净的现代设计，不使用导航截图或古旧地图；
- 地图标注保持极少，并遵守可见文字规则；
- 邮票使用与城市直接相关的小型图像，不编造邮政机构名称或无依据面值；
- 圆形邮戳内写明已经确定的具体日期；
- 邮票、邮戳和地图不得出现乱码、假英文或未经指定的第二种语言。

## 组装顺序

把最终提示词按以下顺序组织成一个连贯段落：

1. 可见文字语言和准确呈现要求；
2. 画幅、背景、边框、质感和配色；
3. 所有可见文字的确切内容；
4. 版式位置；
5. 地标拼贴及主题元素；
6. 地图、邮票、邮戳、坐标；
7. 负面约束和整体气质。

不要留下任何占位符。仅在需要校准信息密度或措辞时读取 [references/examples.md](references/examples.md)；示例只用于结构参考，不得照抄事实或文案。

## 输出前检查

内部确认：

- 城市、国家、地标、坐标和日期没有张冠李戴；
- 季节窗口没有被描述成唯一官方日期；
- 用户参数已正确覆盖默认值；
- 成图中的可见文字只使用选定语言，且所有文案已经写实；
- 地标名称具体，数量一致；
- 中部明确是写实自由拼贴，不是照片墙或插画；
- 地图、邮票、邮戳、日期、坐标、标语和诗性短句完整；
- 没有占位符、解释、引用、分析或自检过程。

## 输出格式

除必要澄清外，最终只输出一段完整提示词。不要输出标题、参数表、调研记录、自检结果、多版本选项或补充说明。
