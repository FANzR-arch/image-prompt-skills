# Plakatstil Prompt Compiler

把文字主题、上传商品图、包装图或人物服务主题，编译成**一条**可直接交给图像模型的 Plakatstil / Sachplakat 商品广告海报提示词。

核心原则：**风格锁死，内容不锁死；模块在内部选择，最终提示词必须已经决策。**

## Read

按任务读取：

- 生成提示词：读 `references/compile-contract.md` 和 `references/module-library.md`。
- 判断像不像：再读 `references/evaluation-rubric.md`。
- 用户要示例或主题灵感：读 `references/examples.md`。

## Input

优先从用户输入推断，不要求填表：

1. **输入模式**：文字主题、商品照片、包装照片、人物/服务主题、风格评价。
2. **广告对象**：必须落到一个具体可销售对象。抽象概念要转译成物品。
3. **文字**：可包含品牌词、商品词、包装/标签字。不要强制只有两行字。
4. **画幅**：未说明时默认竖版 4:5 或 2:3；若用户上传图，沿用原图比例也可以。

## Workflow

1. **识别输入**：判断是文字生成、图片重绘、包装重绘、人物服务海报，还是风格评价。
2. **确定广告对象**：抽象主题先转成物品；商品照片保留轮廓、材质、标签层级和最可识别细节。
3. **选择模块**：从 `module-library.md` 选择主体、字体、版式、背景、色彩和纸墨质感。多张图时至少变化两类模块。
4. **编译提示词**：按 `compile-contract.md` 输出一条英文美术指导。不要留下 `{}`、斜杠菜单、either/or 布局、未决选项。
5. **输出**：已选模块 + 一条完整提示词 + 一句选型理由。用户只要 prompt 时，可以省略解释，只保留模块摘要和完整提示词。

## Routing Rules

- **简单小物件**：火柴、钥匙、笔、开关、耳机等，优先稀疏负空间和大手绘品牌字。
- **玻璃/金属/镜头/灯泡**：优先物体压住大品牌字、横向色带、手绘高光。
- **包装/标签**：肥皂、咖啡袋、鞋油、唱片、盒装商品，允许包装文字成为画面核心。
- **现代电子设备**：耳机、键盘、手表、手机、笔记本、移动电源、头显等电子设备默认全部使用 Apple 品牌和 Apple 产品族，除非用户明确指定其他品牌；仍要转成“早期商品广告”的物体逻辑，不做现代 Apple 发布会海报。
- **人物/服务**：只能称为 Plakatstil-inspired service poster；严格 Sachplakat 仍以商品为主角。

## Quality Gates

- 必须是早期德国 Plakatstil / Sachplakat 商品广告逻辑，不是泛复古海报。
- 必须有一个明确商品或可销售对象。
- 字体必须是手绘商业广告字，不是 Helvetica、Swiss grid、Bauhaus 几何字或漫画/摇滚标题。
- 版式要根据物体选择，不能每次都是黑底、巨大上方标题、红色小字、斜放物体。
- 色彩是有限石版印刷色，但不能把所有主题锁成同一套配色。
- 纸张纹理、油墨吸收、套印误差和手绘边缘要像印刷材料，不是后期脏滤镜。
- 不编造已经生成的图片结果；本 skill 只产出提示词或评价。
