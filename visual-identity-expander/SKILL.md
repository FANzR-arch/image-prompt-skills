# Visual Identity Expander

把一张核心参考图扩展成一套视觉语言统一、可以继续迭代的视觉身份方案。

核心原则：**先锁定可识别的视觉基因，再扩展使用场景。每张图可以变化，但必须让人看出它们属于同一个人、角色、品牌或产品。**

## Read

执行前读取：

- `references/visual-dna.md`：判断不同输入必须保留什么。
- `references/prompt-contract.md`：把分析结果编译成完整提示词。
- 用户需要参考示例或输入过于抽象时，再读 `references/examples.md`。

## Input

优先从用户输入和上传图片推断，不强制填表：

1. **视觉母体**：Logo、人物头像、个人形象、IP、插画、产品或物件。
2. **名称与文字**：品牌名、人名、账号名、口号；没有就不编造。
3. **用途**：个人品牌、商业品牌、内容账号、产品宣传、活动或作品集。
4. **气质**：从参考图推断；用户明确指定时，以用户要求为准。
5. **画幅**：按平台和用途决定；用户指定比例时严格遵循。

只有缺失信息会显著改变结果时才提问。其他内容直接做合理假设，并在输出顶部简短说明。

## Workflow

1. **识别输入类型**：确定是 Logo、人物/头像、IP/插画、产品/物件，或多种输入组合。
2. **提取视觉 DNA**：拆成不可变特征与可变特征。不可变特征用于维持一致性，可变特征用于场景扩展。
3. **建立系统规则**：确定主色、辅助色、形状语言、材质、光线、排版、镜头和情绪。
4. **规划五类输出**：
   - 品牌或个人主视觉
   - 视觉识别系统展示板
   - 角色动作、图形变化或产品变化系统
   - 产品、包装、周边或数字载体应用
   - 社交媒体与真实传播场景
5. **编译提示词**：每类输出生成一条完整、独立、可直接使用的提示词。不要输出未决选项、斜杠菜单或空白占位符。
6. **一致性检查**：逐条检查身份、轮廓、颜色、材质、文字和视觉气质有没有漂移。
7. **按需生图**：如果环境提供图像生成工具且用户明确要求直接生成，按五类输出逐张生成；否则只交付提示词。

## Adaptive Routing

- **Logo**：保留几何比例、负空间、字标关系和品牌色；第三类输出改为辅助图形、纹样与 Logo 变化系统。
- **人物头像/个人形象**：保留脸部身份、发型、眼镜、标志服装和气质；第三类输出生成动作与表情系统。
- **IP/插画角色**：保留轮廓、五官比例、配色和标志性道具；第三类输出生成角色设定与姿态表。
- **产品/物件**：保留外形、材质、结构和品牌标记；第三类输出生成视角、细节与活动主题变化。
- **多张参考图**：先区分主参考与辅助参考。主参考锁身份，辅助参考只提供风格、材质或场景，不混合成新身份。

## Quality Gates

- 五条提示词必须能独立使用，不能依赖“同上”或前文省略信息。
- 每条都要明确引用上传图作为身份锚点，并重申必须保留的特征。
- 不把“画得好看”当成一致性；先检查身份和系统规则，再检查美感。
- 不为 Logo、包装或服装编造无法确认的文字。需要准确文字时，明确写出用户提供的原文。
- 人物延展不随意改变脸型、年龄、族裔、发型和标志性配饰。
- IP 延展不随意改变轮廓、五官比例、配色和标志性道具。
- 产品延展不随意改变结构、接口、标签层级和核心材质。
- 不伪造已经生成的结果。没有调用生图工具时，只声明已完成提示词。
- 避免五张图像五个互不相关的风格案例；变化必须发生在统一规则之内。
