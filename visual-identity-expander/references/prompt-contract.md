# Prompt Contract

每条最终提示词必须是独立的美术指导，至少包含以下部分。

## 1. Reference Anchor

- 明确使用上传图片作为核心参考。
- 写出必须保留的身份、轮廓、颜色、文字或产品结构。
- 区分“内容参考”和“风格参考”。

## 2. Output Goal

- 明确当前生成的是主视觉、识别板、变化系统、应用系统或传播场景。
- 写明用途和画幅。

## 3. Visual System

- 主色与辅助色。
- 几何、线条、纹样或图形语言。
- 字体行为与文字层级。
- 材质、光线、镜头和整体气质。

## 4. Composition

- 主体位置、尺度、裁切、视角、负空间和信息层级。
- 展示板要规定面板数量和排列逻辑；场景图要规定主体与环境关系。

## 5. Text Control

- 只使用用户提供的准确文字。
- 没有文字时明确“no invented text”或“use no additional copy”。
- 对文字准确性要求高时，优先减少文字数量。

## 6. Negative Constraints

根据输入类型加入身份漂移、颜色漂移、随机文字、额外 Logo、结构改变、风格混杂等负面约束。

## 编译要求

- 最终提示词不保留 `{}`、`[]`、斜杠菜单、either/or 或“可选择”。
- 不使用“同上一张”“保持之前风格”等跨提示词省略。
- 如果用户没有指定风格，依据参考图作出一个明确决定，并在“本次假设”中说明。
- 五条提示词共享同一组不可变特征，但构图和使用场景必须有明显区别。
