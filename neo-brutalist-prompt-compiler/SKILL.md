# Neo-Brutalist Prompt Compiler（新粗野主义提示词编译器）

把文章主题、产品介绍或开源项目，编译成**一条**可直接交给图像模型的新粗野主义提示词。

核心原则：**风格锁死，方向先判。** 新粗野主义不是"故意做乱"，是被精确控制的粗糙：边框多粗、阴影多硬、色块怎么撞，都是设计决策。skill 先判断输入属于哪个方向，再按该方向的模板编译。

## Read

- 生成提示词：读 `references/compile-templates.md`（三方向模板 + 风格锁 + 段落规则）。
- 用户要案例或主题灵感：再读 `references/examples.md`（四条已决策成品 prompt + 真实案例清单）。

## Input

优先从用户输入推断，不要求填表：

1. **方向**：文章封面（A）、产品/工具视觉（B）、开发者/开源项目视觉（C）。模糊时默认 A。
2. **主标题**：用户给了原文照抄，不翻译不润色；中文标题建议 2-4 字，最多一行小副标题。
3. **视觉隐喻**：把主题转译成一个组件感图形——按钮、Toggle、终端窗口、卡片堆、断裂网格、标签块。一张图一个主隐喻，不做完整 dashboard。
4. **画幅**：优先用户指定；封面默认横版 16:9 或 5:2。

## Workflow

1. **判方向**：文章/观点 → A；产品官网/界面/Pricing → B；开源项目/API/教程 → C。
2. **B/C 先提醒**：产品截图和代码卡片一旦被识破是假的，会毁掉"这个工具很诚实"的第一印象。B/C 方向默认建议优先引用真实案例（Gumroad、RetroUI、Panda CSS、真实 GitHub 项目，清单在 `examples.md`），AI 生图只做明确标注的**概念示意图**。用户确认要生成后再编译。
3. **选隐喻**：从主题提取一个 UI 组件隐喻。抽象概念要落到具体组件：自动化 → Toggle 开关；方法论 → 终端输入卡；资产 → 卡片堆；趋势本身 → 断裂网格。
4. **编译提示词**：按 `compile-templates.md` 对应方向的模板输出已决策成品，锁住风格、标题、主隐喻和负面约束。
5. **输出**：已选方向 + 一条完整提示词 + 一句选型理由。用户只要 prompt 时省略解释。

## Quality Gates

- 粗边框是结构显形，不是装饰；硬阴影（右下偏移、无羽化）让组件变成物体。
- 扁平强色，主动避开蓝紫渐变 SaaS 默认值；不用玻璃拟态、软阴影、光泽 3D。
- 反精致不等于混乱：表面可以粗，结构不能乱，标题、组件和主视觉关系必须能读出来。
- 一张图一个主隐喻；信息层级多、要讲多步流程的主题不适合这个风格，直接告诉用户。
- 不生成看起来像真实可读代码的内容；开发者方向用抽象块状文字线代替代码。
- 不编造假 UI 微文案、假标签；中文标题必须用用户原文，不出现假中文字。
- 本 skill 只产出提示词或方向建议，不编造已生成的图片结果。
